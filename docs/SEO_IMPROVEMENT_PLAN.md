# SEO抜本改善計画書 - ai-media.co.jp

**作成日**: 2026-03-07
**対象サイト**: https://ai-media.co.jp (AIコンパス by ALLFORCES)
**現状**: 2,103記事 / 月間112PV / Organic Search 18セッション

---

## 1. 診断結果サマリー

### 1.1 致命的な問題

| 問題 | 深刻度 | 影響 |
|------|--------|------|
| OGP画像未設定 2,006件 (95.4%) | **致命的** | SNS共有時にクリックされない、検索結果での視認性ゼロ |
| 孤立記事 1,926件 (91.6%) | **致命的** | Googleがサイト構造を理解できない、クロール効率が極端に低い |
| excerpt短すぎ 1,323件 (62.9%) | **高** | meta descriptionが貧弱、検索結果でのCTR低下 |
| 重複タイトル 23件 | **中** | keyword cannibalization（自サイト内でSEO競合） |
| 内部リンク設定済みわずか5件 (0.2%) | **致命的** | PageRankが分散されず、ほぼ全記事がクロール到達不能 |

### 1.2 良い点（維持すべき）

- `_config.yml`: jekyll-seo-tag, jekyll-sitemap プラグイン導入済み
- `_layouts/default.html`: meta description, OGP, canonical URL, JSON-LD structured data 全て実装済み
- `robots.txt`: 適切な設定（ブロックなし）
- `permalink`: `/:year/:month/:day/:title/` 形式（SEOフレンドリー）
- GA4、Google AdSense 導入済み
- 滞在時間138秒（コンテンツ品質は良好）

### 1.3 根本原因分析

**なぜ2,103記事で月間PV112なのか？**

1. **インデックスされていない可能性が高い**: 91.6%の記事が孤立（内部リンクなし）。Googlebotはリンクを辿ってページを発見するため、サイトマップに記載されていても内部リンクのない記事は優先度が下がり、クロール・インデックスされにくい。
2. **OGP画像なし = SNSからの流入ゼロ**: 95.4%の記事にOGP画像がないため、X/Facebook等でシェアされてもテキストリンクのみで目立たない。
3. **検索結果でのCTR低下**: excerpt（meta description）が短すぎる記事が63%あり、検索結果での説明文が不十分。
4. **重複タイトル**: 同一タイトルが複数あると、Googleが重複コンテンツと判断してインデックスを避ける。
5. **被リンクゼロ**: 外部サイトからのリンクがほぼない（Referralが最大流入源だが26セッション）。

## 2. robots.txt / sitemap 確認結果

### robots.txt
問題なし。全コンテンツがクロール許可されている。

### sitemap
- `jekyll-sitemap`プラグインにより自動生成（GitHub Pages/ビルド時）
- `robots.txt`内に `Sitemap: https://ai-media.co.jp/sitemap.xml` が記載済み
- `sitemap_index.xml` も存在（sitemap.xmlへの参照）
- **確認事項**: ビルド後のsitemap.xmlが全2,103記事を含んでいるか実機確認が必要

### 対応
1. Google Search Consoleでsitemap.xmlを再送信
2. ビルド後のsitemap.xmlの記事数を確認
3. Bing Webmaster Toolsにも登録（BingSiteAuth.xmlは既存）

## 3. 改善ロードマップ

### Phase 1: 緊急修正（1週間以内）

#### 3.1 Google Search Console サイトマップ再送信
```
手順:
1. https://search.google.com/search-console にログイン
2. プロパティ「ai-media.co.jp」を選択
3. 左メニュー「サイトマップ」
4. URL「https://ai-media.co.jp/sitemap.xml」を入力して「送信」
5. インデックスカバレッジレポートで「有効」ページ数を確認
6. 「有効」が2,103より大幅に少ない場合はインデックス問題あり
```

#### 3.2 重複タイトルの修正
- 23件の重複タイトルを手動修正
- `python scripts/fix_seo_meta.py` で対象記事を特定
- 各記事のタイトルに日付や具体的キーワードを追加して差別化

#### 3.3 excerpt改善
- 1,323件のexcerptが短すぎる（50文字未満）
- `python scripts/fix_seo_meta.py --fix` で本文冒頭から自動生成

### Phase 2: 内部リンク構築（2-4週間）

#### 3.4 内部リンク自動追加
```bash
# まずレポート確認
python scripts/add_internal_links.py --report

# 修正実行
python scripts/add_internal_links.py --fix
```
- 1,926件の孤立記事に関連記事リンクを追加
- カテゴリ・タグベースの自動マッチング
- 各カテゴリのピラー記事（最も充実した記事）を特定して集中リンク

#### 3.5 カテゴリページの最適化
| カテゴリ | 記事数 | 対応 |
|---------|--------|------|
| 業界別AI活用 | 946件 | サブカテゴリ分割が必要 |
| AI最新ニュース | 664件 | 月別アーカイブページ追加 |
| AI技術ガイド | 378件 | 技術トピック別インデックスページ |
| 導入事例 | 49件 | 業界別ランディングページ |
| 研究論文 | 38件 | 学会・年度別整理 |
| AI導入戦略 | 28件 | ステップ別ガイド化 |

### Phase 3: コンテンツ品質向上（1-3ヶ月）

#### 3.6 OGP画像の自動生成
- 既存スクリプト `scripts/generate_ogp_image.py` を全記事に適用
- 記事タイトル + カテゴリアイコンのテンプレートベース生成
- 優先度: 最新記事 → 高PV記事 → 全記事

#### 3.7 ピラー記事戦略
各カテゴリに1-2本の「決定版」記事を作成:
- **5,000字以上**の網羅的コンテンツ
- 内部リンクのハブとして機能
- 主要キーワードで上位表示を狙う
- 例: 「AIエージェント完全ガイド2026」「製造業AI導入ロードマップ」

#### 3.8 被リンク獲得
- note.com記事からブログへの相互リンク
- X投稿にブログURLを含める
- はてなブックマーク対策（技術記事のシェア）
- 業界メディアへの寄稿・プレスリリース

## 4. X/noteからの導線強化

### 4.1 X投稿戦略
- 毎日の記事投稿時にブログURLを含める
- 形式: 「[要約1-2行] 詳細はこちら: [URL] #AI #テクノロジー」
- スレッド投稿: 記事の要点を3-5ツイートでまとめ、最後にブログリンク
- リプライやリツイートで既存フォロワーのエンゲージメントを獲得

### 4.2 note.com連携
- note記事の末尾に「より詳しい技術解説はブログで: [URL]」を追加
- noteの無料記事 → ブログの詳細記事への誘導パターン
- 月次まとめ記事をnoteに投稿し、個別記事へのリンクをブログに集約

### 4.3 はてなブックマーク対策
- 技術的に深い記事をはてブで拡散
- タイトルを「〇〇完全ガイド」「〇〇まとめ2026」形式にしてブックマーク誘発
